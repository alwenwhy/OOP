# Распределённая проверка простых чисел

Для заданного массива целых чисел определить, есть ли в нём хотя бы одно
**составное** число. Вычисление распределяется по нескольким узлам одной
подсети через TCP-сокеты.

## Запуск распределённой версии

Откройте **три терминала** в корне проекта.

**Запуск мастера**
```bash
./gradlew runMaster
```

**Запуск слейва**
```bash
./gradlew runSlave
```

Ожидаемый вывод в терминале мастера:
```
Ожидание подключения слейвов...
Слейв подключён: 1
Слейв подключён: 2
...
Слейв подключён: N
Содержит составное: да/нет 
```

Чтобы запустить на разных машинах - замените `"localhost"` в на IP мастера и убедитесь, что нужный порт открыт в подсети.

---

### Топология сети

**Звезда**: один мастер - N слейвов. Мастер - единственный координатор,
слейвы пассивно ждут заданий и возвращают результаты.

После запуска происходит что-то такое:
```
1) Мастер отправляет задание слейву 1
2) Слейв 1 возвращает результат мастеру
3) Мастер отправляет задание слейву 2
4) Слейв 2 возвращает результат мастеру
...
2N - 1) Мастер отправляет задание слейву N
2N) Слейв N возвращает результат мастеру
2N + 1) Мастер отправляет сигнал STOP слейвам и подытоживает ответ
```

### Отказоустойчивость

`SlaveHandler` устанавливает таймаут чтения сокета - 5 секунд (`setSoTimeout`). Если слейв перестаёт отвечать, выбрасывается `SocketTimeoutException`, которое передаётся вызывающему коду.

Для явной проверки доступности слейва реализован метод `sendHeartbeat()`: мастер отправляет сообщение типа `HEARTBEAT`, слейв немедленно отвечает тем же типом. Это позволяет обнаружить зависший узел и можно использовать для перераспределении нагрузки. При недоступности слейва `sendHeartbeat()` истекает по тому же таймауту сокета и выбрасывает исключение.

### Идентичность и состояние

- Каждый `Task` имеет уникальный `taskID`.
- Каждый `Result` содержит `taskID` - мастер может сопоставить ответы с запросами даже при переходе к асинхронному протоколу.
